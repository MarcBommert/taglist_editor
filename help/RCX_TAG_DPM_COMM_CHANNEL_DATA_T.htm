Redefining DPM communication channels

This tag allows the user to modify the RX_HIF_CHANNEL_Ts and the RX_HIF_CHANNEL_BLOCK_Ts describing communication channels: the number of communication channels used and the sizes of the input/output images of each channel.

Number of comm. channelsnumber of communication channels to be instantiated (1 .. DPM_MAX_COMM_CHANNELS)
Channel n Input Area sizeTotal size of the normal priority input data area in bytes
Channel n Output Area sizeTotal size of the normal priority output data area in bytes

Notes:

  • The I/O data size values REPLACE the statically defined values at system startup time, i.e. they serve as ABSOLUTE VALUES. The values are also used for calculating the start offsets of the subsequent channels and blocks.
  • The size of an I/O data area must be a multiple of 4 bytes.
  • The size of an I/O area may be 0. The maximum size is 27904.
  • I/O areas of 11520 bytes or above can only be used with a 32K DPM.
  • The total size of each communication channel is 0x1000 bytes + input area size + output area size.
