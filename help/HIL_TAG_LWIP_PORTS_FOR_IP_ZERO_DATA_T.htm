LWIP Ports for usage with IP 0.0.0.0

With this tag, you can set IP ports which the LWIP stack integrated in firmware shall handle even if it does not have a valid IP configuration. These ports will be usable with Multicast communication only.

Setting a port to 0 means that this entry in taglist shall be ignored. This allows e.g. only activating a single port (or even zero ports).

By default (if taglist is unchanged) the ports shall be set to 0.

Port 0
  • 0: Ignore this entry
  • 1-65535: Set the port number for Port 0.
Port 1
  • 0: Ignore this entry
  • 1-65535: Set the port number for Port 1.
