Socket API Quantity Structure

In this tag you can configure quantity structures related to the Socket API on the Network Services DPM Channel.

Note: Due to limited memory resources of some firmware implementations, restrictions apply regarding the possible values such that the allowed value range may not be useable.

A general hint which combinations are working can not be given here as this is product specific. However, increasing the values (compared to default) will lead to increased memory requirements of the firmware. Value combinations that work for a specific firmware release may no longer work for future releases (in case of memory restricted devices).

The values set here do not influence the quantity structure of resources related to the protocol stack integrated in the firmware. These are independent from the values of the Network Service DPM Channel.

Number of Socket API Services at DPM level The number of Socket API services that shall be handled in parallel at DPM mailbox level.
Allowed values are 1 to 8.
Number of sockets for Socket API isage The number of IP sockets of integrated IP stack that the application wants to use in parallel.
Allowed values are 1 to 64.
