LibStorage Position

If a LibStorage partition is used, this tag can be used to specify the offset and the size of the partition. The LibStorage position is not used within the 2nd Stage Boot Loader. The parameters are handed over to the firmware by using Boot Tokens.

Note: The LibStorage partition has to be used in combination with the "Disk Position" and "Backup Partition" tags.

Attention: The Second Stage Boot Loader does not check for overlapping areas.
Make sure the flash file system (disk position), backup partition and the LibStorage position do not overlap. Also, the last flash sector might be occupied by a Flash Device Label (FDL).

Offset The offset of the LibStorage partition from the beginning of the flash device in bytes.
The offset must be a multiple of the block/sector size of the used flash chip.
Size The size of the LibStorage partition in bytes.
If set to 0, no LibStorage area is available.
The size must be a multiple of the block/sector size of the used flash chip.
The size includes two blocks/sectors which are reserved for the management area.

Example:
The Flash block size is 4096 Byte.
The Offset is 0x20000, which is aligned to the used Flash block size.
The remanent area size is defined with 14 Flash blocks.
The management area needs 2 additional Flash blocks.
Therefore the size for the LibStorage area has to be 16 * 4096 = 65536 bytes.
