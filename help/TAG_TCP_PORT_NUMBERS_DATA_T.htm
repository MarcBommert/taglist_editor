TCP Port Numbers

This tag is used to specify whether incoming TCP/UDP frames are handled by the TCP/IP stack or the Ethernet Interface module.
Usually, unless the Ethernet Interface module is used, the TCP/IP stack handles all incoming TCP/UDP frames by itself. If the Ethernet Interface module is used, the destination port number of a frame determines whether it is handled by the TCP/IP stack or forwarded to the Ethernet Interface module.
This tag defines the ports which are to be handled by the TCP/IP stack. It specifies a range of ports and a list of additional port numbers.

TCP/UDP Port Range (First Port) The first port of the TCP/UDP-Port range. The minimum value is 1024.
TCP/UDP Port Range (Last Port) The last port of the TCP/UDP-Port range. The range must contain at least 1024 ports, i.e. last port - first port >= 1023.
Number of Protocol Stack Ports This is the number of entries from the beginning of the port list which are used by the the protocol stack.
These entries are required by the protocol stack. They are shown for information only and must not be changed.
Number of Additional Ports This is the number of valid entries which have been added by the user.
