Web server (HTTPS) configuration

This tag configures web server's HTTPS related parameters.

Web server (HTTPS) Port Number The TCP port number used by the web server for HTTPS.

The default value is 443.
Range: 0 - 65535

If the port number is set to 0, the web server will deactivate the HTTPS port.

Note: Changing the port to a number that is already used by another component inside the firmware,
will lead to undefined firmware behavior and must be avoided (check the firmware's data sheet for already used TCP ports).
Certificate Handling The Certificate Handling parameter defines the web server's behavior regarding the Security Configuration (Private Key, Default Certificate).
  • 0: Certificate and private key must be present in the Certificate Database, otherwise HTTPS won't start.
  • 1: Self signed certificate and private key will be generated by the web server itself, if any security configuration issue in the Certificate Database is detected during startup.
  • 2: Self signed certificate and private key will be generated by the web server itself, if no configuration within the Certificate Database is available.
  • 3-255: Reserved for future use.
