netFIELD PROFINET IO-Link Profile Configuration Flags

Enable/disable the following special features in the PNSIOLM package.
Note: This replaces the deprecated tag "PROFINET IO-Link Profile Pin4 DIO in IOLM Submodule"

Pin4 DIO data in IOLM Submodule Disabled: Pin4 DIO data in regular profile DI/DO submodules (default)
Enabled: Pin4 DIO data in IOLM Submodule. Profile DI/DO Submodules do not have any IO data in this case.
General mapping of IO-Link events into PROFINET diagnosis Enabled: Profile diagnosis mapping is enabled. The mapping can still be deactivated for individual ports via the PortConfiguration record (default)
Disabled: Profile diagnosis mapping is disabled. The profile still provides the parameter "enable/disable port diagnosis" within the PortConfiguration record. However, the parameter is only stored in the PND to IOLM Proxy - Port Configuration object. The application is responsible for the diagnosis mapping.
General mapping of IO-Link events into PROFINET process alarms Enabled: Profile process alarms mapping is enabled. The mapping can still be deactivated for individual ports via the PortConfiguration record (default)
Disabled: Profile process alarms mapping is disabled. The profile still provides the parameter "enable/disable port process alarms" within the PortConfiguration record. However, the parameter is only stored in the PND to IOLM Proxy - Port Configuration object. The application is responsible for the process alarm mapping.
