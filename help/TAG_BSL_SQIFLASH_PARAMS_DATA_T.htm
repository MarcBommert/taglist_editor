Custom SQI Flash Parameters

The TAG_BSL_SQIFLASH_PARAMS tag can be used to specify a custom SQI flash that is not supported out of the box. It must be used in conjunction with the custom serial flash tag.

Attention: The used flash must conform to the SQIROM unit inside the netX chip (e.g. max wait cycles of 7 clocks, no more than 3 Address bytes, etc.).

Note: Custom SQI flash configuration is available in Second Stage Boot Loader V1.5.0.0 and later.

Note: A special firmware that uses the flash parameters from the Second Stage Boot Loader must be used.

Enable Enable custom SQI parameters
Max. frequency in MHz Maximum Clock frequency in MHz (must be within netX Limits)
Number of address bytes Number of address bytes (netX only supports 3 address bytes)
Command to enter 0-4-4 mode Command to enter 0-4-4 mode
Number of mode clocks Number of mode clocks
Number of dummy cycles Number of dummy cycles
Quad Enable Type Quad Enable Type (see Quad Enable Requirements in SFDP Specification for more details)
1: QE is Bit1 in status register 2 (word accessible)
2: QE is Bit6 in status register 1
3: QE is Bit7 in status register 2
4: QE is Bit1 in status register 2 (byte accessible)
5: QE is Bit1 in status register 2 (split transaction)
Sequence to enter 0-4-4 mode Sequence to enter 0-4-4 mode
1: Send 0xA5 as last address bytes
2: Set 0xA5 in configuration register
Sequence to exit 0-4-4 mode Sequence to exit 0-4-4 mode
1: Send 0x00 at end of current read
2: Send 0xFF as address (8/10 cycles, depending on address width)
