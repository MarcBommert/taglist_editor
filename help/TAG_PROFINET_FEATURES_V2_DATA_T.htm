PROFINET Features V2

In this tag, you can enable, disable or configure some PROFINET features.
Note: Due to limited memory resources of some firmware implementations, restrictions apply regarding the possible combinations of the features. Please refer to PROFINET IO Device API Manual for more details about the consequences of changes to these values.

Number of additional IO Connections (ARs) 0: only 1 cyclic Profinet connection is possible
Allowed values are 0 - 7, please refer to technical data in PROFINET IO Device API Manual for capabilities of the stack version used (it may support less than 7 additional ARs).
IO Supervisor communication accepted Selects whether IO Supervisor communication is accepted by the firmware.
IRT Communication accepted Selects whether IRT communication is accepted by the firmware.
Number of parallel Read Implicit Requests The amount of parallel acyclic Read Implicit Requests that can be handled by the firmware simultaneously.
Allowed values are 1 - 8, please refer to technical data in PROFINET IO Device API Manual for capabilities of the stack version used (it may support less than 8 parallel Services).
MinDeviceInterval The MinDeviceInterval according to the GSDML file of the product.
Allowed values: Power of two in range [8 - 4096]
Number of configurable submodules The amount of submodules an application can configure in the firmware. This number has impact on the runtime memory required by the firmware.
Allowed values are 5 - 1000, please refer to technical data in PROFINET IO Device API Manual for capabilities of the stack version used (it may support less than 1000 submodules).
Size of each RPC buffer in KB The size in KB of each existing RPC buffer preallocated by the firmware. Each AR (IO, DeviceAccess, Supervisor, ReadImplicit) has its own buffer of this size.
Allowed values are 4 - 64, please refer to technical data in PROFINET IO Device API Manual for capabilities of the stack version used.
