PROFINET Features V2

In this tag, you can enable, disable or configure some PROFINET features.
Note: Due to limited memory resources of some firmware implementations restrictions apply regarding the possible combinations of the features. Please refer to PROFINET IO Device API Manual for more details about the consequence of changing one of these values.

Number of configurable submodules The amount of submodules an application can configure in the firmware. This number has impact on the runtime memory required by the firmware.
Allowed values are 1 - 1000.
Please refer to technical data in PROFINET IO Device API Manual for the capabilities of the stack version used (it may support less than 1000 submodules).
Minimum size of each RPC buffer in KB The minimum size in KB of each existing RPC buffer preallocated by the firmware.
Each AR (IO, DeviceAccess, Supervisor, ReadImplicit) has its own buffer of this size. The real buffer size depends on other factors and may be higher than this value.
Allowed values are 4 - 64.
Please refer to technical data in PROFINET IO Device API Manual for the capabilities of the stack version used.
Number of additional IO Connections (ARs) The number of additional cyclic PROFINET IO connections accepted by the firmware. This parameter relates to the Shared Device feature.
Allowed values are 0 - 7:
  • 0: Only 1 cyclic PROFINET connection is possible. Shared Device is not supported.
  • 1-7: In total, 2-8 cyclic PROFINET connections are supported. Shared device is supported.
Please refer to technical data in PROFINET IO Device API Manual for the capabilities of the stack version used (it may support less than 7 additional ARs).

The value of this parameter has an influence on the GSDML file provided with the product. The GSDML parameter "NumberOfAR" needs to be set to the value of this parameter plus 1.

Special handling for firmware including System Redundancy:
  • Setting this parameter to 0 still allows using S2 redundancy with 2 redundant IO-Controllers. Shared Device with 2 non-redundant IO Controllers is not supported in this case. This represents an IO Device supporting S2 redundancy but without Shared Device.
    In this special case, the GSDML parameter "NumberOfAR" needs to be set to value 2.
  • Setting this parameter to 1 (or higher) activates Shared Device plus S2 redundancy in the firmware.
    In this case, the default rule for GSDML parameter "NumberOfAR" applies and needs to be set to this value plus 1.
Number of parallel Read Implicit Requests The amount of parallel acyclic Read Implicit Requests that can be handled by the firmware simultaneously.
Allowed values are 1 - 8.
Please refer to technical data in PROFINET IO Device API Manual for the capabilities of the stack version used (it may support less than 8 parallel Services).
Number of parallel DeviceAccess ARs The amount of supported DeviceAccess ARs.
Allowed values are 0 - 1.
0 means DeviceAccess AR is not supported.
Number of available diagnosis buffers The total amount of buffers the firmware shall reserve for Channel diagnosis datasets.
The application can not create more diagnosis datasets than this number.
Allowed values are 0 - 1000.
